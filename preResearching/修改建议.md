Type I：Unified End-to-End Foundation Models

主线保留

RT-2：作为 Type I 的起点最合适，因为它最清楚地展示了把 semantic understanding、reasoning 和 action generation 放进一个统一 VLA 架构里。
OpenVLA：保留。它把 Type I 从闭源示范推进到开源、可微调、可部署的 7B VLA 形态。
π0：保留。它不是普通 follow-up，而是 Type I 从离散 token action 转向连续 flow-matching control 的关键一步。
GR00T N1：保留，但写成 Type I 的边界代表。它仍是统一 foundation model，但已经显式具有 dual-system flavor，正好说明 Type I 的上限和向 Type II 的过渡。

侧提

X-VLA：建议一句带过，作为 Type I 的 cross-embodiment 扩展。它本质上仍是统一 VLA，只是通过 embodiment-specific soft prompts 强化异构数据利用，不值得挤进四篇主讲。
ReconVLA：可以一句带过，因为它代表 Type I 开始引入显式 3D reconstructive bias。
UniVLA / NORA-1.5 / Interleave-VLA / RDT-2：建议留在表格，不要逐篇展开。主线已经足够。

移走

这一类都应从 Type I 正文主体移到 deployment / efficiency：DeeR-VLA, MoLe-VLA, VLA-Cache, BitVLA, VLA-Adapter, TinyVLA, SmolVLA, Edge VLA, Evo-1, OpenVLA-OFT。这些是 serving、compression、adaptation efficiency 议题，不是 Type I 的 primary architectural commitment。
Type II：Hierarchical & Modular Reasoning Models

主线保留

RT-H：保留。它最清楚地定义了 Type II 的早期形态：通过 language motions 建立高层—低层 hierarchy。
FiS-VLA：保留。它是 slow semantic branch + fast execution branch 这条主线最直接的代表。
CoT-VLA：保留。它说明 Type II 不只是分层，还在显式引入 intermediate reasoning trace 来桥接 action。
RoBridge：强烈建议保留。它几乎是 Type II 最贴题的一篇：high-level cognitive planner + symbolic bridge + embodied agent，直接把“bridging cognition and execution”作为主旨。

侧提

ThinkAct：可以侧提，但不建议和 CoT-VLA 同时重讲；二者功能高度相近。
Hi Robot / HAMSTER / GO-1：各用一句带过，分别承担 open-ended feedback、cross-embodiment hierarchy、工业规模层级系统的扩展角色。
GR00T N1：如果已在 Type I 主讲，这里只保留一句边界说明，不要重复展开。

移走

OK-Robot：建议移出 Type II 主体。它是 modular zero-shot OVMM/open-knowledge system，更像 embodied system case，而不是你这里 manipulation integration taxonomy 的主线代表。
SafeVLA：移到 deployment / safety。
MemoryVLA / GraphCoT-VLA / Cosmos-Reason1 / A0 / Maestro / ControlVLA：全部降成一句群组化，别占主线空间。
Type III：Spatial Grounding & Predictive World Models

主线保留

3D-VLA：保留。它最明确地把 3D perception、reasoning 和 action 通过 generative world model 连起来，适合作为 Type III 的起点。
SpatialVLA：保留。它代表“显式空间状态作为耦合界面”，通过 Ego3D Position Encoding 和 Adaptive Action Grids 强化 spatial representation。
UWM：强烈建议保留。它不是一般 world model，而是同一 transformer 兼作 policy、forward/inverse dynamics 和 video predictor，非常适合作为 Type III 主轴。
GR-2：保留。它代表 web-scale video dynamics pretraining 如何反过来塑造 manipulation agent，是 Type III 的另一条重要路数。
BridgeVLA：保留，但放在 Type III 末尾作为收束项。它通过 input-output alignment 把 3D manipulation 的空间接口统一起来，比很多 3D 方法更像真正的 integration bridge。

侧提

PointVLA：建议一句带过。它更像 3D injection module，而不是 Type III 的定义性 work。
DreamVLA：可一句带过，作为 world-model-enhanced VLA 的例子。
V-JEPA 2：只保留一句，因为它更自然属于前面 B5 的 latent predictive model 脉络，不应在 Type III 主体重讲。

移走

RoBridge：从 Type III 表格移回 Type II；它本质上是 hierarchical symbolic bridge，不是 world-state coupling。
这些建议整体移出 Type III 主体：3D Diffuser Actor, EquiGraspFlow, 3DAffordSplat, PhysTwin, TraceVLA, TrackVLA, RoboRefer, RoboTracer, MolmoAct, VideoVLA, Gen2Act, Dreamitate, Dream2Real, UVA, VITA, ViPRA, LAPA, Moto, CoWVLA, Tactile-VLA, 3D-ViTac, AnySkin, ManiWAV。理由很一致：它们更像 3D representation / tracking / affordance、video pretraining、policy parameterization，或 multimodal sensing extension，而不是 Type III 的 primary architectural commitment。
